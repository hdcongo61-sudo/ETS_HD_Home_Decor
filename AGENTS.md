# AI Design Rules for This App

## Main Goal
Improve the app design to feel premium, mobile-first, clean, and professional.

## Design Style
Use a design style inspired by:
- Apple
- Instagram Threads
- Linear
- Notion
- Modern premium ecommerce apps

## UI Rules
- Mobile-first layout
- Clean spacing
- Rounded corners: 16px to 24px
- Soft shadows
- Glass blur where useful
- Smooth animations
- Beautiful empty states
- Skeleton loaders
- Clear buttons
- Elegant cards
- Sticky mobile navigation when needed
- Dark mode and light mode friendly

## React Rules
Use:
- React
- Tailwind CSS
- Framer Motion
- lucide-react
- Recharts for charts

## Important
Do not break existing logic.
Do not remove API calls.
Do not change backend routes.
Only improve UI, UX, responsiveness, spacing, animations, and component structure.

## Anti-AI-Generated Design Rule

This app must NOT look AI-generated.

Avoid:
- Generic gradient hero sections
- Random glassmorphism everywhere
- Too many icons
- Oversized cards
- Fake SaaS landing-page style
- Repetitive layouts
- Unnecessary animations
- Generic blue/purple gradients
- Same-looking buttons everywhere
- Empty decorative UI with no purpose

Design like a real product designer:
- Use restraint
- Use strong spacing
- Use fewer but better visual elements
- Use real hierarchy
- Make each screen functional first, beautiful second
- Use consistent typography
- Use practical mobile layouts
- Make buttons, forms, tables, and cards feel intentional
- Use animations only when they help the user
- Prefer clean product UI over flashy AI UI

Style direction:
- Human-made
- Premium
- Practical
- Mobile-first
- Ecommerce-ready
- Inspired by Apple, Threads, Linear, and high-end African commerce apps

Act as a senior frontend architect.

Rules:
- Create scalable reusable components
- Separate UI, logic, hooks, and services
- Keep components modular
- Use loading skeletons
- Add proper empty states
- Add error handling UI
- Use responsive breakpoints correctly
- Optimize for mobile performance
- Use clean Tailwind utility patterns
- Avoid duplicated code
- Prefer maintainable architecture over fast hacks

Structure:
- components/ui
- components/dashboard
- hooks
- services
- utils
- layouts
- pages

Every page must look polished and enterprise-grade.

Add premium UI polish like modern SaaS apps.

Requirements:
- Smooth page transitions
- Stagger animations
- Hover states
- Skeleton loaders
- Floating navbar
- Blur backgrounds
- Animated gradients
- Premium button interactions
- Elegant empty states
- Sticky mobile bottom navigation
- Animated charts
- Real-time feeling interface

Use:
- Framer Motion
- CSS transitions
- Transform animations
- Opacity animations
- Scale effects
- Spring animations

Animations must feel smooth and subtle like Apple products.

Never generate generic AI UI.
Every interface must feel designed by a real senior product designer.
Use visual hierarchy, premium spacing, and intentional layouts.
